UI based repository handling:- The XML based repository used for Descriptive Programming is managed using a GUI. This takes away the burden of having to hand-edit the XML file.

Data validation:- The data entered by the user while editing the XML Repository is validated. This eliminates errors that arise due to wrong values being entered into the XML Repository.

Mixed repository support:- The Repository Manager allows you to describe an object either using name-value attributes (as in true descriptive programming) or pick an object straight off Quicktest Pro’s object repository.

Import Quicktest Repository:- One of the most powerful features of the Repository Manager is its ability to import from a Quicktest shared repository. When this feature is excercised, each object definition from the shared repository is converted to a corresponding ‘descriptive programming’ object and appended to the XML Repository that is currently being edited.

Easy transition for new users:- A combination of all the above features makes the transition to using an ‘XML based repository for descriptive programming’ a smooth one.

For those who are already using descriptive programming, the UI based management feature provides an easy way to create and manage the XML repository. The data validation feature helps in reducing various types of errors.

For those who have never used descriptive programing, but have been wanting to make the shift, the Repository Manager provides an easy transition path. In addition to the UI based management and the data validation features, the Quicktest repository import feature provides a segue into descriptive programing world.
