EMOS Framework Core

EMOS Framework is a simple yet powerful environment for development of automated WinRunner® tests. This package contains the core libraries required to run system.

Getting Started

  • To get EMOS Framework started you need to load the libraries from this package. You will normally do this from the startup script in your WinRunner project. All you need to do is to set the search path to this directory and to call the "emosint" test. This test will load all libraries that are needed by EMOS Framework. You can see an example of such startup test in "Hello World" example.
  • There is an (mostly incomplete) history document with changes that were made on each version. To see what's new in this release check the top link in this document.
  • There is an old User's Guide (or something that was once intended to serve that purpose). Although this document is a bit outdated you should still read it because all main concepts still apply.
  • There is also an API documentation for the EMOS_GPL package. Although you don't need to know much about it it is not a bad idea to get acquainted with core libriaries such as emos_frm_lib, emos_frm_gui_lib and emos_frm_stp_lib.
  • To start with EMOS Framework you should download the "Hello World" example and follow the instructions in readme.html. Note that we used to call this project "TPL" because it usually serves as a template for new projects. In fact, you can still find it under EMOS_GPL/FRM/TPL.
  • For other examples consult EMOS Framework home page.
